# 并行爬虫库任务清单（搜索引擎抓取场景）

## 现状结论
- [ ] 主流程未闭环：`Start()` 尚未驱动批量抓取、链接扩展与结果落盘
- [ ] 媒体下载能力部分可用：已支持按 `Content-Type` 识别并保存图片/音频/视频
- [ ] 正则预处理仅有配置字段：未进入执行链路
- [ ] 关键治理能力缺失：去重、robots、并发调度、结构化结果存储

## P0（先修错误）
- [ ] 修复 `Reprocess` 的 JSON tag 拼写错误（`josn` -> `json`）
- [ ] 修复 `(*WebConfig).Add()` 先解引用后判空导致的空指针风险
- [ ] 修复 `RequestWebPage` 对 `webConf` 的空指针保护
- [ ] 修复重试循环里非 2xx 响应未及时关闭 `resp.Body` 的连接泄漏
- [ ] 统一 `UserData/userdata` 路径大小写，保证跨平台一致
- [ ] 修复 URL 文件名解析在空路径下可能出现 `"."` 的问题
- [ ] 为配置反序列化增加错误处理，避免静默失败

## P1（补齐核心能力）
- [ ] 实现抓取调度主流程：种子 URL 队列 -> 请求 -> 解析 -> 新链接入队
- [ ] 实现 HTML 链接提取与归一化（绝对化、去锚点、去重复参数策略）
- [ ] 实现域名范围控制与深度控制（同域优先、最大深度、最大页面数）
- [ ] 实现已访问去重（URL 规范化 + 集合/布隆过滤器）
- [ ] 实现文本提取流水线（HTML 清洗、正文提取、编码统一）
- [ ] 实现正则预处理流水线（提取、替换、过滤规则可配置）
- [ ] 实现 robots.txt 规则读取与访问判定
- [ ] 实现并发与限速策略（worker pool、域名级限速、随机抖动）
- [ ] 落实反爬策略执行（代理轮换、UA 轮换、失败退避策略）

## P2（存储与可观测性）
- [ ] 设计结构化结果模型（页面、媒体、来源链接、抓取时间、状态）
- [ ] 支持结果持久化（JSONL/SQLite 二选一，建议先 JSONL）
- [ ] 记录失败重试明细与错误类型分布
- [ ] 增加任务统计指标（成功率、下载量、去重率、平均耗时）
- [ ] 输出可追踪日志字段（任务ID、域名、URL、重试次数）

## P3（工程质量）
- [ ] 为核心链路补充单元测试（配置、重试、链接解析、去重、预处理）
- [ ] 为下载与解析流程补充集成测试（本地静态站点样例）
- [ ] 清理未使用代码与占位模块（`SmartStart`、`downloadFile`、空模块）
- [ ] 增加配置校验器（冲突项、缺失项、非法值自动告警）
- [ ] 提供最小可运行示例配置（搜索引擎结果页抓取模板）

## 验收清单
- [ ] 输入搜索结果页后，可自动抓取并扩展子链接
- [ ] 可分别保存文本、图片、音频、视频，并保留来源关系
- [ ] 正则预处理可配置且实际生效
- [ ] 同域限速与 robots 规则生效，避免高风险访问
- [ ] 长时间运行无明显连接泄漏与内存异常增长
